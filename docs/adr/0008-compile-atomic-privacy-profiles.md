# Compile atomic privacy profiles

Each consumer pack supplies one immutable privacy profile plus narrow product
adapters, and `helto-privacy` atomically validates and compiles that profile into
typed server and browser handles for one fixed privacy contract suite. This
preserves discoverable workflow, record, artifact, and execution interfaces
while keeping policy non-configurable, registration order-independent, and
browser/server state fingerprinted; a generic command port was rejected as too
opaque, and a consumer-assembled capability graph was rejected as unnecessary
interface breadth and version drift for the coordinated five-repository cutover.
