# Gate release readiness on complete synthetic evidence

A five-repository suite becomes `ready` only when one signed evidence manifest,
bound to its immutable suite digest, proves every versioned acceptance-catalog
obligation for each exact supported Python/ComfyUI/frontend/renderer tuple.
`helto-privacy` owns genuine historical-writer fixtures made from synthetic
inputs, the shared contract and leak-oracle harness, all 24 registration orders,
negative blocked-install cells, deterministic fault campaigns, and disposable
rendered scenarios; consumers contribute real adapters and product semantics.
User workflows, media, keys, browser state, and decrypted values are excluded
from release evidence, while local installation verification remains
operator-blind and separate from explicit activation. Skips, waivers,
retry-to-green results, inferred version ranges, consumer-local privacy test
engines, live-data smoke tests, and Director's vendored fallback were rejected
because they would let the manifest claim privacy behavior that was not proved.
